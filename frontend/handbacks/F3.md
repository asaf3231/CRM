## 4. Decisions made

**Type shape decisions:**
- `ReachPoint` — added 6 named fields (formBot, connect, email1, email2, liMessage, email3) replacing the old 5-field shape. The old type was only partially used (no component consumed it); the rename is non-breaking.
- `AgentEvent` — replaced `ts`/`kind`/`detail` with `date`/`time`/`text` to support the day-divider grouping pattern. The old shape had no consumer.
- `Cohort` — fully replaced to carry `variants[]` (each with `stages[]` and `outcome`). The old `stages`/`variant`/`enrolledAt` single-variant shape is gone; the new shape is a superset.
- `CohortVariantStage.icon` — typed as `"clock"|"linkedin"|"mail"` union (not a React component type) to keep the data layer free of React imports.
- `OutreachStats.inCampaign` + `inCampaignCohorts` — added `inCampaignCohorts` for the "across N cohorts" sub-line. Old `inProgress` renamed to `inCampaign` for clarity.
- `EnrollmentEvent` — new type: `{ id, date, label }`.

**Layout decisions:**
- Middle row and bottom row both use `grid grid-cols-3` (left panel `col-span-2`, right panel `col-span-1`) — matches the reference's ~2/3 : 1/3 split.
- CohortTimeline's stage row uses `overflow-x-auto` on the inner strip (not on the Card), so the card border doesn't clip but wide timelines scroll horizontally.
- Day-dividers in AgentEventLog use `sticky top-0 bg-card` so they pin while the list scrolls.
- Calendar "today" highlight uses `bg-muted` on the cell; mock enrollments are in May 2026 so they show up when the calendar opens on today's month.
