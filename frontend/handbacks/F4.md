## 4. Decisions made

- **`FlowNodeStatus` exported as a named type** from `FlowNode.tsx` so both feature files can import it without re-declaring the union. This is the minimal-friction pattern matching the existing codebase style.
- **`FlowNode` accepts `children`** so `SwarmStageCard` can inject the progress bar + pixel chips without creating a separate wrapper that duplicates the idle/running/done ring logic. Keeps the card behavior in one place.
- **Timer architecture: flat ref array** (`pushTimer`) rather than a Map or object — the cleanup only needs `forEach(clearTimeout)` and there's no need to cancel individual timers mid-run. Simple, correct.
- **Second-run dedup** is done client-side in both `api.runDiscovery` (filters `MOCK_DISCOVERY_RESULTS_2` by domains not in `MOCK_DISCOVERY_RESULTS`) and in the component (`seen = new Set(results.map(r => r.domain))`). Belt-and-suspenders.
- **`Tavily Recovery` subtitle** uses `fires when &lt; 2 results` (HTML entity) in JSX so the `<` doesn't trigger a lint/parse warning. In React JSX inside a string prop it would be fine, but I used it in a JSX text context.
- **`overallProgress` update inside the Analyzer's tick loop** sets `pixelActive` and `processed` in two separate `setRuntime` calls. To avoid a race, the pixel-chip update is co-located inside the same tick timer as the `processed` update — they fire together, just two separate `setRuntime` calls. This is safe because React batches state updates in the same event loop turn (React 18 automatic batching).
- **`/outreach` Placeholder updated** from `stage="F4"` to `stage="F5"` — it was the only remaining F4 placeholder and is not in scope for F4 (brief says "Leave `/outreach` (Outreach Engine) as Placeholder").

## 5. Deviations from this brief — with reason

- **Light theme preserved** — brief explicitly states "build LIGHT screens, inside the existing light app shell." Done. Reference slides are dark marketing slides; we use our own design tokens throughout.
- **`FlowNode` tone coloring uses CSS variable ring colors** (`ring-info/40`, `ring-warning/40`) rather than hardcoded hex — this matches the token-based approach of the existing codebase.
- **No explicit `LucideIcon` passed to `FlowNode` in the fanout diagram** — the brief's FlowNode spec says `icon?` is optional. Omitting it on the pipeline nodes keeps them cleaner (the title and subtitle are the signal); the icon slot is available for future use.
- **`totalDomains` in the Swarm run-bar** is the average of all stage `targetCount` values (all are 100 in the mock) rather than a hardcoded "100" — avoids a magic number inline.

## 6. Risks / things for the PM to check in the preview

1. **3-way fanout layout** — the fan-out arrow column uses a `flex-col self-stretch` container with three `flex-1` rows. If the three vector-node column is taller than expected, the top/bottom arrows may not visually align with the topmost/bottommost node. Check at 1280px and 1440px.
2. **Streaming timing** — with `results` captured in the `run` useCallback closure, a second consecutive rapid run (clicking Run Again before the first fully finishes) is guarded by `if (running) return`. Verify that clicking "Run Again" immediately after results appear works without stale closure issues — `runCount` and `results` are both in the dependency array of `useCallback`.
3. **Swarm pixel chip animation** — uses `Math.ceil((t / tickCount) * 3)` to reveal chips progressively. At tickCount=8 this gives: t=1→1 chip, t=3→2 chips, t=5→2, t=6→3 chips. Verify visually that all 3 chips are revealed before the Analyzer stage marks "done".
4. **Overall progress bar overshoot** — `overallBase` uses `Math.max(overallBase, overallStep)` to prevent backward jumps; the final `setOverallProgress(100)` sets it to exactly 100 after the totalDuration timer. Check that there's no visual jitter on the last stage.
5. **Swarm `useEffect` dependency lint** — `stages.length` is used as the dep for the runtime initialization effect (with a suppression comment) to avoid re-initializing on every render. The actual `stages` array reference changes on every TanStack Query re-render. This is intentional and correct — but a lint runner that ignores the suppression comment may flag it.
